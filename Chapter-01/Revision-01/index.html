Basics

HTML Essentials: The Skeleton

(WHAT) is it?

The structure of your web page. We use Tags like <form>, <input>, and <button> to build the user interface.

(WHY) do we use it?

Without HTML, the browser wouldn't know where to put the "Student Name" box or the "Submit" button. It's the layout of our camp!

(HOW) did we do it?

We used the name="..." attribute on inputs so PHP can identify them later. <input type="text" name="student_name">
Basics

PHP Essentials: The Brain

(WHAT) is it?

A server-side language that thinks for us. It starts with <?php and ends with ?>.

(WHY) do we use it?

HTML is just a static picture. PHP makes it "alive" by calculating grades, total marks, and averages based on what the user typed.

(HOW) did we do it?

We used Variables (starting with $) to store data like $name = $_POST['name'];. <?php echo "Hello, " . $name; ?>

Syntax Spotlight: Variables

$userName = "Dilan";
  • $: In PHP, every variable name must start with a dollar sign.
  • =: This doesn't mean "equals" here. It means "put the value on the right into the box on the left".
Lesson 01

HTML Forms & Basic PHP

(WHAT) is it?

The very first step: Creating a UI for users to enter a student name and their mark.

(WHY) do we use it?

To learn how the web communicates data from a browser to a server.

(HOW) did we do it?

We used the <form> tag with two critical attributes:
  • action="process.php": This is the Destination. It tells the browser "When the user clicks submit, send this to process.php".
  • method="POST": This is the Delivery Method. POST hides the data inside the request (secure and invisible to the user).
  • name="mark": This is the Label. PHP uses this label to find the value ($_POST['mark']).

Syntax Spotlight: Conditionals

if ($mark >= 75) { ... }
  • if (...): PHP's way of asking a question.
  • { ... }: The curly brackets are "Arms". They hold all the code that should run ONLY if the answer is YES.
Lesson 02

Loops & Multiple Subjects

(WHAT) is it?

Scaling up to handle multiple marks for one student.

(WHY) do we use it?

Because manual processing is slow. Arrays let us treat 5 marks as ONE group.

(HOW) did we do it?

By adding square brackets [] to the input names: name="marks[]".

This tells HTML to pack all those inputs into an Array. In PHP, we then use a foreach loop to "visit" every mark one-by-one to calculate the total.

foreach ($marks as $single_mark) { // visits every item }

Syntax Spotlight: Foreach Loop

foreach ($marks as $val) { ... }
  • $marks: The whole collection (The list of marks).
  • as $val: A nickname for the current item we are looking at.
  • How it works: PHP takes the first mark, calls it $val, runs your code, then loops back to take the second mark, and so on.
Lesson 03

The Full Classroom

(WHAT) is it?

Processing many students at once using dynamic forms.

(WHY) do we use it?

To build scalable apps. Imagine a teacher entering grades for 40 students in one go.

(HOW) did we do it?

We used Nested Arrays. Each student has their own "box":
  • marks[0]: Folder for Student 1.
  • marks[1]: Folder for Student 2.

We used Nested Loops (a loop inside a loop) — the outer loop picks a student, and the inner loop calculates their marks.

Syntax Spotlight: Deep Access

$allMarks[0]['scores'][1]
  • [0]: Go to the FIRST Student's folder.
  • ['scores']: Look inside the box labeled 'scores'.
  • [1]: Pick the SECOND mark in that box.
Lesson 04

Modular Functions

(WHAT) is it?

Moving calculation logic into a special file called `functions.php`.

(WHY) do we use it?

So we don't repeat code. If you want to calculate grades on 10 different pages, you only write the function ONCE.

(HOW) did we do it?

We used the include statement. Think of it as "copy-pasting" all the code from functions.php into your script automatically.

Then we called our custom function getLetterGrade($score), which takes a number and "returns" a letter (A, B, C...).

Syntax Spotlight: Functions

function sum($a, $b) { return $a + $b; }
  • sum: The name of the machine.
  • ($a, $b): The Inputs (Ingredients) the machine needs to work.
  • return: This pushes the result OUT of the machine so you can use it.
Lesson 05

Ranking & Sorting

(WHAT) is it?

Comparing student averages to award ranks (#1, #2, #3).

(WHY) do we use it?

To provide insight into classroom performance and top-scorers.

(HOW) did we do it?

We used the usort() function with a special comparison rule.

It's like telling PHP: "Hey, here is a list of students. Use this special rule (Compare Average B to Average A) to put the smartest student at the top of the list."

// Comparison rule: Highest Average first
return ($b['avg'] > $a['avg']) ? 1 : -1;

Syntax Spotlight: The Ternary Operator

($b['avg'] > $a['avg']) ? 1 : -1;
  • (The Parentheses): This is the Question. "Is Student B's average higher than Student A's?"
  • The ?: This means "Then". If the answer is YES, return the next value (1).
  • The :: This means "Else". If the answer is NO, return the last value (-1).
  • Result: If it returns 1, PHP swaps their positions in the list, effectively putting the higher grade on top!
